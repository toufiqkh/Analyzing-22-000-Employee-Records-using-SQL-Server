Human Resources Data Analysis (SQL & People Analytics)

📌 Project Overview
This project involves a comprehensive end-to-end data analysis of a corporate Human Resources dataset containing over 22,000 employee records. Using SQL Server (T-SQL), I performed extensive data cleaning, transformation, and exploratory data analysis (EDA) to uncover critical workforce insights regarding demographics, tenure, and turnover.

🛠️ Tech Stack & Skills
Database: Microsoft SQL Server (T-SQL)

Key Techniques: * Data Cleaning: Handling inconsistent date formats (MM-DD-YY vs MM/DD/YYYY) and managing NULL values.

Advanced SQL: CTEs (Common Table Expressions), Subqueries, Window Functions, and Case Statements.

Analysis: Workforce Demographics, Geographic Distribution, and Retention Analysis.

📁 Dataset Description
The dataset includes 13 columns covering employee identification, personal demographics, and professional details:

Employee Info: id, first_name, last_name

Demographics: birthdate, gender, race

Job Details: department, jobtitle, location, hire_date, termdate

Geography: location_city, location_state

🔍 Key Insights & Questions Answered
Workforce Demographics: Identified the gender and racial breakdown across the organization.

Age Distribution: Grouped employees into brackets (21-30, 31-40, 41-50, etc.) to understand the company’s age maturity.

Departmental Breakdown: Analyzed which departments (e.g., Engineering, Accounting) have the highest headcount.

Retention & Turnover: Calculated the average tenure of employees and identified departments with high termination rates.

Geographic Spread: Mapped employee distribution across multiple states (with Ohio being the primary hub).
